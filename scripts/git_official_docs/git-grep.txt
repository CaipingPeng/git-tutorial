[p] git-grep - Print lines matching a pattern

[pre] git grep[-a | --text] [-I] [--textconv] [-i | --ignore-case] [-w | --word-regexp]
	   [-v | --invert-match] [-h|-H] [--full-name]
	   [-E | --extended-regexp] [-G | --basic-regexp]
	   [-P | --perl-regexp]
	   [-F | --fixed-strings] [-n | --line-number] [--column]
	   [-l | --files-with-matches] [-L | --files-without-match]
	   [(-O | --open-files-in-pager) [<pager>]]
	   [-z | --null]
	   [ -o | --only-matching ] [-c | --count] [--all-match] [-q | --quiet]
	   [--max-depth <depth>] [--[no-]recu

[h2] DESCRIPTION

[p] Look for specified patterns in the tracked files in the work tree, blobs
registered in the index file, or blobs in given tree objects.  Patterns
are lists of one or more search expressions separated by newline
characters.  An empty string as search expression matches all lines.

[p] Instead of searching tracked files in the working tree, search
blobs registered in the index file.

[p] In addition to searching in the tracked files in the working
tree, search also in untracked files.

[p] Search files in the current directory that is not managed by Git,
or by ignoring that the current directory is managed by Git.  This
is rather similar to running the regulargrep(1)utility with its-roption specified, but with some additional benefits, such as
using pathspec patterns to limit paths;  see thepathspecentry
ingitglossary[7]for more information.

[p] This option cannot be used together with--cachedor--untracked.
See alsogrep.fallbackToNoIndexinCONFIGURATIONbelow.
