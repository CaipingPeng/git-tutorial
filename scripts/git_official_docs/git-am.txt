[h2] DESCRIPTION

[p] Splits mail messages in a mailbox into commit log messages,
authorship information, and patches, and applies them to the
current branch. You could think of it as a reverse operation
ofgit-format-patch[1]run on a branch with a straight
history without merges.

[p] The list of mailbox files to read patches from. If you do not
supply this argument, the command reads from the standard input.
If you supply directories, they will be treated as Maildirs.

[p] Add aSigned-off-bytrailer to the commit message (seegit-interpret-trailers[1]), using the committer identity
of yourself.  See the signoff option ingit-commit[1]for more information.

[p] Pass-kflag togit-mailinfo[1].

[p] Pass-bflag togit-mailinfo[1].

[p] With--keep-cr, callgit-mailsplit[1]with the same option, to prevent it from stripping CR at the end of
lines.am.keepcrconfiguration variable can be used to specify the
default behaviour.--no-keep-cris useful to overrideam.keepcr.

[code] --no-keep-cr

[p] Remove everything in body before a scissors line (seegit-mailinfo[1]). Can be activated by default using

[p] This flag will be passed down togit-mailinfo[1].

[p] How to handle an e-mail message lacking a patch:

[p] The e-mail message will be skipped.

[p] An empty commit will be created, with the contents of the e-mail
message as its log.

[p] The command will fail, stopping in the middle of the currentamsession. This is the default behavior.
