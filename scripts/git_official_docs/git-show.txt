[p] git-show - Show various types of objects

[pre] gitshow[<options>] [<object>…​]

[h2] DESCRIPTION

[p] Shows one or more objects (blobs, trees, tags and commits).

[p] For commits it shows the log message and textual diff. It also
presents the merge commit in a special format as produced bygitdiff-tree--cc.

[p] For tags, it shows the tag message and the referenced objects.

[p] For trees, it shows the names (equivalent togitls-treewith--name-only).

[code] --name-only

[p] For plain blobs, it shows the plain contents.

[p] Some options thatgitlogcommand understands can be used to
control how the changes the commit introduces are shown.

[p] This manual page describes only the most frequently used options.

[p] The names of objects to show (defaults toHEAD).
For a more complete list of ways to spell object names, see
"SPECIFYING REVISIONS" section ingitrevisions[7].

[p] Pretty-print the contents of the commit logs in a given format,
where<format>can be one ofoneline,short,medium,full,fuller,reference,email,raw,format:<string>andtformat:<string>.  When<format>is none of the above,
and has%<placeholder>in it, it acts as if--pretty=tformat:<format>were given.

[code] --pretty=tformat:

[p] See the "PRETTY FORMATS" section for some additional details for each
format.  When=<format>part is omitted, it defaults tomedium.

[code] --abbrev-commit

[p] Instead of showing the full 40-byte hexadecimal commit object
name, show a prefix that names the object uniquely.--abbrev=<n>(which also modifies diff output, if it is displayed)
option can be used to specify the minimum length of the prefix.

[p] This should make--pretty=onelinea whole lot more readable for
people using 80-column terminals.

as--oneline. It also overrides thelog.abbrevCommitvariable.

[code] --encoding=

[p] Commit objects record the character encoding used for the log message
in their encoding header; this option can be used to tell the
command to re-code the commit log message in the encoding
preferred by the user.  For non plumbing commands this
defaults to UTF-8. Note that if an object claims to be encoded
inXand we are outputting inX, we will output the object
verbatim; this means that invalid sequences in the original
commit may be copied to the output. Likewise, if iconv(3) fails
to convert t

[p] Perform a tab expansion (replace each tab with enough spaces

[p] With an optional<ref>argument, use the ref to find the notes

[p] Multiple--notesoptions can be combined to control which notes are

[code] --notes=foo

[p] Do not show notes. This negates the above--notesoption, by

Options are parsed in the order given on the command line, so e.g.
"--notes--notes=foo--no-notes--notes=bar" will only show notes
