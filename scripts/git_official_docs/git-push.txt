[h2] DESCRIPTION

[p] Updates one or more branches, tags, or other references in a remote
repository from your local repository, and sends all necessary data
that isn’t already on the remote.

[p] The simplest way to push isgitpush<remote><branch>.gitpushoriginmainwill push the localmainbranch to themainbranch on the remote namedorigin.

ororiginif there’s no configured upstream.

[p] Specify what destination ref to update with what source object.

[p] The format for a refspec is[+]<src>[:<dst>], for examplemain,main:other, orHEAD^:refs/heads/main.

[code] HEAD^:refs/heads/main

[p] The<src>is often the name of the local branch to push, but it can be
any arbitrary "SHA-1 expression" (seegitrevisions[7]).

[p] The<dst>determines what ref to update on the remote side. It must be the
name of a branch, tag, or other ref, not an arbitrary expression.

[p] The+is optional and does the same thing as--force.

[p] You can write a refspec using the fully expanded form (for
examplerefs/heads/main:refs/heads/main) which specifies the exact source
and destination, or with a shorter form (for examplemainormain:other). Here are the rules for how refspecs are expanded,
as well as various other special refspec forms:

[code] refs/heads/main:refs/heads/main
