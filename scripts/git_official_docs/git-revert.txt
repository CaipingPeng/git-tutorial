[p] git-revert - Revert some existing commits

[pre] git revert[--[no-]edit] [-n] [-m <parent-number>] [-s] [-S[<keyid>]]<commit>…​git revert(--continue | --skip | --abort | --quit)

[h2] DESCRIPTION

[p] Given one or more existing commits, revert the changes that the
related patches introduce, and record some new commits that record
them.  This requires your working tree to be clean (no modifications
from the HEAD commit).

[p] Note:git revertis used to record some new commits to reverse the
effect of some earlier commits (often only a faulty one).  If you want to
throw away all uncommitted changes in your working directory, you
should seegit-reset[1], particularly the--hardoption.  If
you want to extract specific files as they were in another commit, you
should seegit-restore[1], specifically the--sourceoption. Take care with these alternatives as
both will discard uncommitted changes in your working directory.

[p] See "Reset, restore and revert" ingit[1]for the differences
between the three commands.

[p] Commits to revert.
For a more complete list of ways to spell commit names, seegitrevisions[7].
Sets of commits can also be given but no traversal is done by
default, seegit-rev-list[1]and its--no-walkoption.

[p] With this option,git revertwill let you edit the commit
message prior to committing the revert. This is the default if
you run the command from a terminal.

[p] Usually you cannot revert a merge because you do not know which
side of the merge should be considered the mainline.  This
option specifies the parent number (starting from 1) of
the mainline and allows revert to reverse the change
relative to the specified parent.

[p] Reverting a merge commit declares that you will never want the tree changes
brought in by the merge.  As a result, later merges will only bring in tree
changes introduced by commits that are not ancestors of the previously
reverted merge.  This may or may not be what you want.

[p] See therevert-a-faulty-merge How-Tofor
more details.

[p] With this option,git revertwill not start the commit
message editor.

[p] This option determines how the commit message will be cleaned up before
being passed on to the commit machinery. Seegit-commit[1]for more
details. In particular, if the<mode>is given a value ofscissors,
scissors will be appended toMERGE_MSGbefore being passed on in the case
of a conflict.

[p] Usually the command automatically creates some commits with
commit log messages stating which commits were
reverted.  This flag applies the changes necessary
to revert the named commits to your working tree
and the index, but does not make the commits.  In addition,
when this option is used, your index does not have to match
the HEAD commit.  The revert is done against the
beginning state of your index.

[p] This is useful when reverting more than one commits'
effect to your index in a row.
