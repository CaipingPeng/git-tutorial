[p] git-request-pull - Generates a summary of pending changes

[pre] git request-pull[-p] <start> <URL> [<end>]

[h2] DESCRIPTION

[p] Generate a request asking your upstream project to pull changes into
their tree.  The request, printed to the standard output,
begins with the branch description, summarizes
the changes, and indicates from where they can be pulled.

[p] The upstream project is expected to have the commit named by<start>and the output asks it to integrate the changes you made
since that commit, up to the commit named by<end>, by visiting
the repository named by<URL>.

[p] Include patch text in the output.

[p] Commit to start at.  This names a commit that is already in
the upstream history.

[p] The repository URL to be pulled from.

[p] Commit to end at (defaults to HEAD).  This names the commit
at the tip of the history you are asking to be pulled.

[p] When the repository named by<URL>has the commit at a tip of a
ref that is different from the ref you have locally, you can use the<local>:<remote>syntax, to have its local name, a colon:, and
its remote name.

[p] Imagine that you built your work on yourmasterbranch on top of
thev1.0release, and want it to be integrated into the project.
First you push that change to your public repository for others to
see:

[pre] git push https://git.ko.xz/project master

[p] Then, you run this command:

[pre] git request-pull v1.0 https://git.ko.xz/project master

[p] which will produce a request to the upstream, summarizing the
changes between thev1.0release and yourmaster, to pull it
from your public repository.

[p] If you pushed your change to a branch whose name is different from
the one you have locally, e.g.

[pre] git push https://git.ko.xz/project master:for-linus

[p] then you can ask that to be pulled with

[pre] git request-pull v1.0 https://git.ko.xz/project master:for-linus

[p] Part of thegit[1]suite
