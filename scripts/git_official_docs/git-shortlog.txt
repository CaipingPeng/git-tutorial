[p] git-shortlog - Summarizegit logoutput

[pre] git shortlog[<options>] [<revision-range>] [[--] <path>…​]
git log --pretty=short |git shortlog[<options>]

[h2] DESCRIPTION

[p] Summarizesgit logoutput in a format suitable for inclusion
in release announcements. Each commit will be grouped by author and title.

[p] Additionally, "[PATCH]" will be stripped from the commit description.

[p] If no revisions are passed on the command line and either standard input
is not a terminal or there is no current branch,git shortlogwill
output a summary of the log read from standard input, without
reference to the current repository.

[p] Sort output according to the number of commits per author instead
of author alphabetic order.

[p] Suppress commit description and provide a commit count summary only.

[p] Show the email address of each author.

[p] Instead of the commit subject, use some other information to
describe each commit.<format>can be any string accepted
by the--formatoption ofgit log, such as* [%h] %s.
(See the "PRETTY FORMATS" section ofgit-log[1].)

[p] Each pretty-printed commit will be rewrapped before it is shown.

[p] Show dates formatted according to the given date string. (See
the--dateoption in the "Commit Formatting" section ofgit-log[1]). Useful with--group=format:<format>.

[h3] Commit Limiting

[p] Besides specifying a range of commits that should be listed using the
special notations explained in the description, additional commit
limiting may be applied.

[p] Using more options generally further limits the output (e.g.--since=<date1>limits to commits newer than<date1>, and using it
with--grep=<pattern>further limits to commits whose log message
has a line that matches<pattern>), unless otherwise noted.

[p] Note that these are applied before commit
ordering and formatting options, such as--reverse.

[code] --max-count=

[p] Limit the output to<number>commits.

[p] Skip<number>commits before starting to show the commit output.

[p] Show commits more recent than<date>.

[code] --since-as-filter=

[p] Show all commits more recent than<date>. This visits
all commits in the range, rather than stopping at the first commit which
is older than<date>.

[p] Show commits older than<date>.

[code] --committer=

[p] Limit the commits output to ones with author/committer
header lines that match the<pattern>regular
expression.  With more than one--author=<pattern>,
commits whose author matches any of the<pattern>are
chosen (similarly for multiple--committer=<pattern>).
