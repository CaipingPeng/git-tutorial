**改进内容**：

**改进前**（问题）：
`markdown
-a 的意思是 "all tracked files"。注意它**不会 add 新文件**（untracked）
`

**改进后**（解决方案）：
`markdown
-a 的意思是 "all tracked files"。它有两个重要限制：

1. **不会 add 新文件**（untracked files）
2. **不会 add 被 .gitignore 忽略的文件**

这意味着如果你：
- 创建了新文件 → 必须先 git add 新文件
- 想精确控制暂存哪些改动 → 不能用 -a

**推荐习惯**：新手阶段建议总是先 git add，再 git commit，这样更容易理解暂存区的作用。
`

**效果**：避免新手误用 -a 导致新文件未提交的常见错误。

**新增内容结构**：
`markdown
### 快照模型的实际意义

**传统差异式**：
版本1 → 版本1+补丁A → 版本1+补丁A+补丁B

**Git 快照式**：
版本1快照 | 版本2快照 | 版本3快照

**这带来什么好处？**

| 操作 | 差异式 | 快照式（Git） |
|-----|-------|-------------|
| 切换到任意版本 | 需要应用一串补丁 | 直接取出快照 |
| 创建分支 | 需要复制文件 | 只需创建指针 |
| 合并分支 | 复杂的补丁合并 | 三方快照合并 |
| 检出速度 | 慢 | 快 |
`

**效果**：帮助读者建立正确的 Git 心智模型，理解分支和合并的本质。
