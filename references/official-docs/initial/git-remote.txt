[h2] DESCRIPTION

[p] Manage the set of repositories ("remotes") whose branches you track.

[p] Be a little more verbose and show remote url after name.
For promisor remotes, also show which filters (blob:noneetc.)
are configured.
NOTE: This must be placed betweenremoteand subcommand.

[p] With no arguments, show a list of existing remotes.  Several
subcommands are available to perform operations on the remotes.

[p] Add a remote named<name>for the repository at<URL>.  The commandgitfetch<name>can then be used to create and
update remote-tracking branches<name>/<branch>.

[p] With-foption,gitfetch<name>is run immediately after
the remote information is set up.

[p] With--tagsoption,gitfetch<name>imports every tag from the
remote repository.

[p] With--no-tagsoption,gitfetch<name>does not import tags from
the remote repository.

[p] By default, only tags on fetched branches are imported
(seegit-fetch[1]).

[p] With-t<branch>option, instead of the default glob
refspec for the remote to track all branches under
therefs/remotes/<name>/namespace, a refspec to track only<branch>is created.  You can give more than one-t<branch>to track
multiple branches without grabbing all branches.

[p] With-m<master>option, a symbolic-refrefs/remotes/<name>/HEADis set
up to point at remote’s<master>branch. See also the set-head command.

[p] When a fetch mirror is created with--mirror=fetch, the refs will not
be stored in therefs/remotes/namespace, but rather everything inrefs/on the remote will be directly mirrored intorefs/in the
local repository. This option only makes sense in bare repositories,
because a fetch would overwrite any local commits.

[code] set-branches

[p] Change the list of branches tracked by the named remote.
This can be used to track a subset of the available remote branches
after the initial setup for a remote.

[p] The named branches will be interpreted as if specified with the-toption on thegitremoteaddcommand line.

[p] With--add, instead of replacing the list of currently tracked
branches, adds to that list.

[p] Retrieves the URLs for a remote. Configurations forinsteadOfandpushInsteadOfare expanded here. By default, only the first URL is listed.

[code] pushInsteadOf

[p] With--push, push URLs are queried rather than fetch URLs.

[p] With--all, all URLs for the remote will be listed.

[p] Change URLs for the remote. Sets first URL for remote<name>that matches
regex<oldurl>(first URL if no<oldurl>is given) to<newurl>. If<oldurl>doesn’t match any URL, an error occurs and nothing is changed.
