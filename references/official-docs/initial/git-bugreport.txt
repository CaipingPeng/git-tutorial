[p] git-bugreport - Collect information for user to file a bug report

[pre] git bugreport[(-o | --output-directory) <path>]
		[(-s | --suffix) <format> | --no-suffix]
		[--diagnose[=<mode>]]

[h2] DESCRIPTION

[p] Collects information about the user’s machine, Git client, and repository
state, in addition to a form requesting information about the behavior the
user observed, and stores it in a single text file which the user can then
share, for example to the Git mailing list, in order to report an observed
bug.

[p] The following information is requested from the user:

[li] Reproduction steps

[p] Reproduction steps

[li] Expected behavior

[p] Expected behavior

[li] Actual behavior

[p] Actual behavior

[p] The following information is captured automatically:

[li] git version --build-options

[p] git version --build-options

[li] uname sysname, release, version, and machine strings

[p] uname sysname, release, version, and machine strings

[li] Compiler-specific info string

[p] Compiler-specific info string

[li] A list of enabled hooks

[p] A list of enabled hooks

[p] Additional information may be gathered into a separate zip archive using the--diagnoseoption, and can be attached alongside the bugreport document to
provide additional context to readers.

[p] This tool is invoked via the typical Git setup process, which means that in some
cases, it might not be able to launch - for example, if a relevant config file
is unreadable. In this kind of scenario, it may be helpful to manually gather
the kind of information listed above when manually asking for help.

[p] Place the resulting bug report file in<path>instead of the current
directory.

[p] Specify an alternate suffix for the bugreport name, to create a file
namedgit-bugreport-<formatted-suffix>. This should take the form of a
strftime(3) format string; the current local time will be used.--no-suffixdisables the suffix and the file is just namedgit-bugreportwithout any disambiguation measure.

[code] git-bugreport

[p] Create a zip archive of supplemental information about the user’s
machine, Git client, and repository state. The archive is written to the
same output directory as the bug report and is namedgit-diagnostics-<formatted-suffix>.

[p] Withoutmodespecified, the diagnostic archive will contain the default set of
statistics reported bygitdiagnose. An optionalmodevalue may be specified
to change which information is included in the archive. Seegit-diagnose[1]for the list of valid values formodeand details
about their usage.
