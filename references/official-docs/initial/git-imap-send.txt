[p] git-imap-send - Send a collection of patches from stdin to an IMAP folder

[pre] git imap-send[-v] [-q] [--[no-]curl] [(--folder|-f) <folder>]git imap-send--list

[h2] DESCRIPTION

[p] This command uploads a mailbox generated withgitformat-patchinto an IMAP drafts folder.  This allows patches to be sent as
other email is when using mail clients that cannot read mailbox
files directly. The command also works with any general mailbox
in which emails have the fieldsFrom,Date, andSubjectin
that order.

[code] format-patch

[p] Typical usage is something like:

[pre] $ git format-patch --signoff --stdout --attach origin | git imap-send

[p] Be verbose.

[p] Specify the folder in which the emails have to saved.
For example:--folder=[Gmail]/Draftsor-fINBOX/Drafts.

[code] INBOX/Drafts

[p] Use libcurl to communicate with the IMAP server, unless tunneling
into it.  Ignored if Git was built without the USE_CURL_FOR_IMAP_SEND
option set.

[p] Talk to the IMAP server using git’s own IMAP routines instead of
using libcurl.  Ignored if Git was built with the NO_OPENSSL option
set.

[p] Run the IMAP LIST command to output a list of all the folders present.

[h2] GETTING A LIST OF AVAILABLE FOLDERS

[p] In order to send an email to a specific folder, you need to know the correct name of
intended folder in your mailbox. The names like "Junk", "Trash" etc. displayed by
various email clients need not be the actual names of the folders stored in the mail
server of your email provider.

[p] In order to get the correct folder name to be used withgitimap-send, you can rungitimap-send--list. This will display a list of valid folder names. An example
of such an output when run on a Gmail account is:

[pre] * LIST (\HasNoChildren) "/" "INBOX"
* LIST (\HasChildren \Noselect) "/" "[Gmail]"
* LIST (\All \HasNoChildren) "/" "[Gmail]/All Mail"
* LIST (\Drafts \HasNoChildren) "/" "[Gmail]/Drafts"
* LIST (\HasNoChildren \Important) "/" "[Gmail]/Important"
* LIST (\HasNoChildren \Sent) "/" "[Gmail]/Sent Mail"
* LIST (\HasNoChildren \Junk) "/" "[Gmail]/Spam"
* LIST (\Flagged \HasNoChildren) "/" "[Gmail]/Starred"
* LIST (\HasNoChildren \Trash) "/" "[Gmail]/Trash"

[p] Here, you can observe that the correct name for the "Junk" folder is[Gmail]/Spamand for the "Trash" folder is[Gmail]/Trash. Similar logic can be used to determine
other folders as well.

[p] Using tunnel mode:

[pre] [imap]
    folder = "INBOX.Drafts"
    tunnel = "ssh -q -C user@example.com /usr/bin/imapd ./Maildir 2> /dev/null"

[p] Using direct mode:

[pre] [imap]
    folder = "INBOX.Drafts"
    host = imap://imap.example.com
    user = bob
    pass = p4ssw0rd

[p] Using direct mode with SSL:

[pre] [imap]
    folder = "INBOX.Drafts"
    host = imaps://imap.example.com
    user = bob
    pass = p4ssw0rd
    port = 123
    ; sslVerify = false

[code] sslVerify=false

[p] Using Gmail’s IMAP interface:

[pre] [imap]
    folder = "[Gmail]/Drafts"
    host = imaps://imap.gmail.com
    user = user@gmail.com
    port = 993

[p] Gmail does not allow using your regular password forgitimap-send.
If you have multi-factor authentication set up on your Gmail account, you
can generate an app-specific password for use withgitimap-send.
Visithttps://security.google.com/settings/security/apppasswordsto create
it. Alternatively, use OAuth2.0 authentication as described below.

[p] If you want to use OAuth2.0 based authentication, you can specifyOAUTHBEARERorXOAUTH2mechanism in your config. It is more secure
than using app-specific passwords, and also does not enforce the need of
having multi-factor authentication. You will have to use an OAuth2.0
access token in place of your password when using this authentication.

[code] OAUTHBEARER

[pre] [imap]
    folder = "[Gmail]/Drafts"
    host = imaps://imap.gmail.com
    user = user@gmail.com
    port = 993
    authmethod = OAUTHBEARER
