# Git 术语中英对照表

本教程使用中文术语，但阅读英文资料时会遇到英文术语。下表帮助你建立对照。

| 中文 | 英文 | 说明 |
|-----|------|------|
| 工作目录 | Working Directory / Working Tree | 你正在编辑的项目文件夹 |
| 暂存区 | Staging Area / Index | 下次提交要包含的改动清单 |
| 仓库 | Repository | 存放版本历史的地方（.git 目录） |
| 提交 | Commit | 一次项目快照 |
| 提交说明 | Commit Message | 描述这次改动的文字 |
| 分支 | Branch | 指向某个提交的可移动指针 |
| 标签 | Tag | 指向某个提交的固定标记，通常用于版本发布 |
| 合并 | Merge | 把一个分支的改动整合到另一个分支 |
| 变基 | Rebase | 把一系列提交移到新的基础提交上 |
| 冲突 | Conflict | Git 无法自动决定如何合并时需要人工判断 |
| 远程仓库 | Remote Repository | 服务器上的仓库（如 GitHub） |
| 源 | Origin | 默认远程仓库的名字 |
| 上游 | Upstream | 你 fork 的原始仓库 |
| 克隆 | Clone | 下载远程仓库到本地 |
| 推送 | Push | 把本地提交上传到远程仓库 |
| 拉取 | Pull | 从远程仓库下载并合并到本地 |
| 获取 | Fetch | 从远程仓库下载但不合并 |
| 拉取请求 | Pull Request (PR) | GitHub/Gitee 术语 |
| 合并请求 | Merge Request (MR) | GitLab 术语 |
| HEAD | HEAD | 当前所在的位置（通常指向当前分支） |
| 快照 | Snapshot | 项目在某一时刻的完整状态 |
| 对象 | Object | Git 内部存储单元（blob、tree、commit、tag） |
| 哈希 | Hash / SHA | 提交的唯一标识符（40位十六进制） |
| 引用 | Reference / Ref | 指向提交的名字（分支、标签等） |
| 远程跟踪分支 | Remote-tracking Branch | 本地对远程分支状态的记录（如 origin/main） |

## 常见命令术语

| 中文 | 英文 | 说明 |
|-----|------|------|
| 暂存 | Stage / Add | 把改动加入暂存区 |
| 取消暂存 | Unstage | 从暂存区移除 |
| 撤销 | Undo / Revert | 取消某个改动 |
| 重置 | Reset | 移动分支指针 |
| 恢复 | Restore | 恢复文件到某个状态 |
| 检出 | Checkout | 切换分支或恢复文件（老命令，现在推荐用 switch 和 estore） |
| 保存现场 | Stash | 临时保存未提交的改动 |
| 变更 | Change / Modification | 文件的改动 |
| 差异 | Diff / Difference | 两个版本之间的不同 |
| 日志 | Log | 提交历史 |
| 责备 | Blame | 查看每行代码最后是谁改的 |
| 精选 | Cherry-pick | 把某个提交的改动应用到当前分支 |
| 二分查找 | Bisect | 用二分法定位引入bug的提交 |

## 状态术语

| 中文 | 英文 | 说明 |
|-----|------|------|
| 未跟踪 | Untracked | 新文件，Git 还没有管理它 |
| 已修改 | Modified | 文件已改动但未暂存 |
| 已暂存 | Staged | 改动已加入暂存区 |
| 已提交 | Committed | 改动已保存到仓库历史 |
| 未修改 | Unmodified | 文件内容和最近一次提交一致 |
| 干净的 | Clean | 工作目录没有未提交的改动 |
| 脏的 | Dirty | 工作目录有未提交的改动 |
| 领先 | Ahead | 本地分支有远程没有的提交 |
| 落后 | Behind | 远程分支有本地没有的提交 |
| 分叉 | Diverged | 本地和远程各有对方没有的提交 |
| 游离 | Detached | HEAD 不指向任何分支 |

## 使用建议

1. **看英文资料时**：对照本表找到对应的中文概念
2. **搜索问题时**：中英文关键词都试一试
3. **团队协作时**：统一使用一套术语，避免混乱
4. **Git 命令提示**：Git 的英文提示通常很有用，学会看懂它们
