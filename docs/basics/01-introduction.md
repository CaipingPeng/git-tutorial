# 01. Git 是什么

> 5 分钟了解为什么要用 Git

## 为什么需要版本控制?

想象你在写论文,文件夹里可能有这些:

```
论文_初稿.docx
论文_修改版.docx
论文_最终版.docx
论文_最终版_真的最终了.docx
论文_最终版_导师修改.docx
论文_这次是最终版.docx
```

这就是手动"版本控制"的混乱场景。

**更糟糕的情况**:
- 😰 想回到两周前的版本,但找不到了
- 😱 多人协作时,不知道谁改了什么
- 💥 合并不同人的修改时出错

Git 就是用来解决这些问题的。

## Git 是什么?

**Git 是一个分布式版本控制系统。**

用人话说:
- **版本控制**: 自动记录文件的每次改动
- **分布式**: 每个人都有完整的历史记录
- **系统**: 一套管理代码变化的工具

## Git 的核心优势

### 1. 自动记录历史

```bash
# 提交一次改动
git commit -m "添加登录功能"

# 想回到任何时候
git checkout 昨天的版本
git checkout 上周的版本
git checkout 任何一次提交
```

不需要手动保存多个版本,Git 自动帮你记录。

### 2. 安全的并行开发

```
主线:  A---B---C---D
             \
功能分支:      E---F
```

- 在分支上开发新功能
- 不影响主线代码
- 开发完再合并

### 3. 轻松协作

```
Alice 的电脑        GitHub        Bob 的电脑
     |               |                |
     |--→ push  --→|                |
     |               |←-- pull ←-----|
     |←-- pull ←-----|                |
     |               |←-- push  ←-----|
```

- 每个人都有完整代码
- 可以离线工作
- 自动合并改动

### 4. 时光机

```bash
# 哦不!删错文件了
git restore 被删的文件

# 哦不!这个提交有bug
git revert 有问题的提交

# 想看谁改了这行代码
git blame 文件名
```

Git 记录了一切,可以随时回溯。

## Git vs 其他工具

| 特性 | Git | SVN (老工具) | 手动备份 |
|---|---|---|---|
| 速度 | 快 (本地操作) | 慢 (需要连接服务器) | - |
| 离线工作 | ✅ 可以 | ❌ 不行 | ✅ 可以 |
| 分支 | 轻量快速 | 笨重缓慢 | 复制整个文件夹 |
| 合并 | 自动智能 | 容易出错 | 手动复制粘贴 |
| 流行度 | ⭐⭐⭐⭐⭐ | ⭐⭐ | ⭐ |

## Git 的核心概念

### 快照 vs 差异

**传统工具** (保存差异):
```
版本1: 完整文件
版本2: +2行, -1行
版本3: +5行, -3行
```

**Git** (保存快照):
```
版本1: 整个项目的快照
版本2: 整个项目的快照
版本3: 整个项目的快照
```

Git 不保存差异,而是保存每个版本的完整快照。

**为什么这样设计?**
- 切换版本超快 (不用计算差异)
- 更可靠 (每个版本都是完整的)
- 压缩存储 (重复内容只保存一份)

### 三个工作区域

```
工作目录          暂存区          本地仓库
  (你编辑代码)     (准备提交)      (永久保存)
     |              |               |
     | git add      |               |
     |------------→|               |
     |              | git commit    |
     |              |-------------→|
```

- **工作目录**: 你看到和编辑的文件
- **暂存区**: 临时存放要提交的改动
- **本地仓库**: 永久保存的版本历史

## 你只需要记住这 3 点

1. **Git 是快照系统**: 每次提交都是完整的项目快照
2. **分布式**: 每个人都有完整历史,可以离线工作
3. **三个区域**: 工作目录 → 暂存区 → 本地仓库
