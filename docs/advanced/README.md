# Git 进阶教程

> **开发中** - 本部分内容正在编写，适合已掌握基础操作、希望深入理解 Git 原理与高级技巧的用户。

## 📚 章节导航

### 1. Git 内部原理
**预计时长**: 20 分钟  
**内容**: 对象模型（blob/tree/commit）、引用系统、reflog、.git 目录结构  
**目标**: 理解 Git 底层存储机制，为高级操作打下基础

### 2. 高级分支管理
**预计时长**: 15 分钟  
**内容**: 分支策略对比（Git Flow/GitHub Flow/Trunk-Based）、长期分支管理、孤立分支  
**目标**: 根据团队规模选择合适的分支策略

### 3. Rebase 精通
**预计时长**: 20 分钟  
**内容**: 交互式 rebase、修改历史提交、合并多个提交、rebase vs merge 决策  
**目标**: 掌握 rebase 的各种场景与风险控制

### 4. 高级合并技巧
**预计时长**: 15 分钟  
**内容**: 合并策略（ours/theirs/octopus）、子树合并、cherry-pick 批量操作  
**目标**: 应对复杂合并场景

### 5. 历史改写与维护
**预计时长**: 15 分钟  
**内容**: filter-branch、filter-repo、BFG Repo-Cleaner、敏感信息清除  
**目标**: 安全地清理仓库历史

### 6. Git Worktree
**预计时长**: 10 分钟  
**内容**: 多工作区并行开发、worktree 与分支的关系、最佳实践  
**目标**: 提升多任务并行效率

### 7. Submodules 与 Subtree
**预计时长**: 15 分钟  
**内容**: 子模块管理、subtree 合并、monorepo vs multirepo  
**目标**: 管理复杂项目依赖

### 8. Git Hooks 实战
**预计时长**: 15 分钟  
**内容**: 客户端钩子（pre-commit/commit-msg）、服务端钩子、Husky 集成  
**目标**: 自动化代码质量检查

### 9. 性能优化
**预计时长**: 10 分钟  
**内容**: gc 与打包、shallow clone、sparse checkout、partial clone  
**目标**: 优化大型仓库的克隆与操作速度

### 10. 高级技巧集锦
**预计时长**: 15 分钟  
**内容**: bisect 调试、blame 溯源、reflog 救援、worktree 技巧、自定义别名  
**目标**: 掌握提升效率的小技巧

---

## 🎯 学习路径建议

1. **按顺序学习**: 前 3 章（内部原理/分支/Rebase）是后续内容的基础
2. **按需选读**: 4-10 章可根据实际工作需求选择性阅读
3. **实践验证**: 建议在测试仓库中实践每个高级命令

## ⚠️ 注意事项

- 本部分内容涉及历史改写等危险操作，请务必先在测试仓库练习
- 修改已推送的公共历史会影响团队协作，需谨慎操作
- 建议先完成[基础教程](../basics/)的学习
