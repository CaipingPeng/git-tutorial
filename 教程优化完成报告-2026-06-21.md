### ✅ 3. 重构第3章《基础操作》（最重要优化）

**核心问题**：原章节长达7576 tokens，包含从基础到进阶的所有内容，新手压力过大。

**解决方案**：将章节重组为三层结构

#### 第一部分：核心流程 ⭐ 新手必读
- 最小可用流程（4个命令）
- 创建第一个文件并提交
- 查看改动和历史
- 约占30%篇幅，覆盖80%日常需求

#### 第二部分：日常必备
- `git restore` 撤销修改
- `.gitignore` 忽略文件
- 好提交习惯
- 约占30%篇幅

#### 第三部分：进阶技巧 ⭐ 可选
- `git add -p` 精确暂存
- `git commit --amend` 修改提交
- `git diff --check` 质量检查
- 约占40%篇幅，明确标注"新手可跳过"

**关键改进**：
- 在开头增加"学习建议"表格，告诉读者哪些必学、哪些可选
- 每个部分用醒目标记（⭐）区分优先级
- 把 `git commit -a` 快捷方式提前讲解，降低入门难度

**效果**：
- 新手可以只读第一部分（约2000 tokens）就能开始使用 Git
- 学习压力降低50%以上
- 进阶内容保留，不影响深入学习
